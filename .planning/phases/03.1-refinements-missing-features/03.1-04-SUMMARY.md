## Decisions Made
- `CATEGORY_COLOR_MAP` keyed by color name string — Tailwind purges dynamic class names at build time; all color classes must be statically present
- `CalendarEvent.category` stays as plain string (category name) with separate `categoryColor`/`categoryIcon` fields — simpler for calendar API responses while enabling rich display
- `CategoryBadge` updated to accept `Category` object; calendar components (list-view, event-popover) inline the badge rendering directly to avoid passing Category objects through CalendarEvent
- `CategoryFilter` changed from hardcoded ALL_CATEGORIES to `categories: Category[]` prop — caller is responsible for fetching and passing categories
