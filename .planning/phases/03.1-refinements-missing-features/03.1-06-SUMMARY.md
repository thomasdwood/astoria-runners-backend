## Decisions Made
- `CalendarEvent` flattened in backend: `formatEventForCalendar` now maps `category.color` → `categoryColor`, `category.icon` → `categoryIcon`, and exposes `routeId`/`startLocation`. This fixes a discrepancy from plan 04 where the frontend type expected flat fields but the backend still returned a nested category object.
- Cancel vs. Delete distinction: Cancel = visible as grey/strikethrough (runners know it's off); Delete = invisible (completely removed). Both are important admin control options.
- Conflict detection uses time-of-day buckets (morning/afternoon/evening) rather than exact overlap — more lenient but avoids false positives from different start times on same day.
- `instanceDefaults` prop on EventForm allows pre-filling route/time/notes from a recurring instance without changing the existing Event-based API.
