**Extended rrule-based recurring template system with weekly/biweekly/monthly nth-weekday support, natural language preview API, excludedDates filtering, and isCancelled flag on calendar responses**

## Accomplishments
- Extended `buildRRule` to accept `RecurrencePattern` object supporting weekly, biweekly (interval: 2), and monthly nth-weekday (bysetpos) patterns — no count ever included
- Added `formatRecurrenceText` producing "Every Monday", "Every other Tuesday", "Every 2nd Wednesday of the month" without rrule.toText()
- Added `getRecurrencePreview` returning natural language text + next 3 dates via 90-day window `.between()`
- Added `GET /api/recurring-templates/preview` endpoint (auth-protected, must come before `/:id`)
- Added `isCancelled` field to `CalendarEvent` interface; propagated from DB event in `formatEventForCalendar`
- Added `excludedDates` filtering in `getAllInstancesInRange` and `getInstancesInRange` (YYYY-MM-DD Set lookup)
- Fixed category filter from `{ category: string }` to `{ categoryId: number }` across recurring service and routes
- Fixed `validateQuery` middleware to assign parsed values back to `req.query` (enables Zod coercion)
- Simplified `errorHandler` response format to `{ error: message }` string

## Decisions Made
- Used `RecurrencePattern` object interface for `buildRRule` instead of individual params — cleaner API and easier to extend
- `formatRecurrenceText` uses local constants (not `rrule.toText()`) because rrule's toText() produces incorrect output for bysetpos patterns
- Preview endpoint is `GET /preview` with query params (idempotent, bookmarkable) placed before `/:id` to avoid Express routing conflict
- `isCancelled: false` for virtual recurring instances (they can't be cancelled until materialized as DB events)
- categoryId integer FK filter replaces category string filter throughout — consistency with 03.1-01 schema migration
