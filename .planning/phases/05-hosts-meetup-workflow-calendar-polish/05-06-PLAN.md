  <files>src/db/seed.ts</files>
  <action>
The seed script's categories upsert uses `target: categories.name` (a Drizzle column reference). Drizzle resolves this to a unique constraint, but the live database may have the constraint under a different name if the table was created outside of the generated migration. The fix is to use the explicit constraint name string `"categories_name_unique"` as the conflict target — this matches the constraint defined in the schema via `.unique()`.

In `src/db/seed.ts`, locate the categories `onConflictDoUpdate` block (lines 76-83) and change `target: categories.name` to use the explicit constraint name:

```typescript
.onConflictDoUpdate({
  target: sql`(name)`,
  set: {
    color: category.color,
    icon: category.icon,
    updatedAt: sql`NOW()`,
  },
})
```

Using `sql\`(name)\`` as the target tells PostgreSQL to use the unique index on the `name` column directly (equivalent to `ON CONFLICT (name) DO UPDATE`), which works regardless of whether the constraint was created with the named `CONSTRAINT "categories_name_unique"` clause or as an implicit unique index. The `sql` import is already present at line 10.

Do not change any other upsert targets in the file — the settings and users upserts already use correct column references (`settings.key`, `users.email`) which have named unique constraints that match.
