    1. Initialize Node.js project:
       - `npm init -y` then update package.json with project name "astoria-runners", type "module"
       - Add scripts: "dev": "tsx watch src/server.ts", "build": "tsc", "db:generate": "drizzle-kit generate", "db:migrate": "drizzle-kit migrate", "db:push": "drizzle-kit push", "db:seed": "tsx src/db/seed.ts"

    2. Install dependencies:
       - Core: `npm install express express-session drizzle-orm @node-rs/argon2 pg helmet express-rate-limit dotenv`
       - Session store: `npm install connect-redis ioredis`
       - Dev: `npm install -D typescript tsx drizzle-kit @types/express @types/express-session @types/pg`

    3. Create tsconfig.json:
       - target: "ES2022", module: "Node16", moduleResolution: "Node16"
       - outDir: "./dist", rootDir: "./src"
       - strict: true, esModuleInterop: true, skipLibCheck: true

    4. Create docker-compose.yml with two services:
       - postgres: image postgres:16-alpine, port 5432:5432, env POSTGRES_DB=astoria_runners, POSTGRES_USER=postgres, POSTGRES_PASSWORD=postgres, volume for data persistence
       - redis: image redis:7-alpine, port 6379:6379

    5. Create .env.example with all required variables (no actual secrets):
       ```
       NODE_ENV=development
       PORT=3000
       DB_HOST=localhost
       DB_PORT=5432
       DB_NAME=astoria_runners
       DB_USER=postgres
       DB_PASSWORD=postgres
       SESSION_SECRET=change-me-to-a-random-string-at-least-32-chars
       REDIS_URL=redis://localhost:6379
       ```

    6. Copy .env.example to .env with the same development defaults.

    7. Create .gitignore: node_modules, dist, .env (but NOT .env.example), *.js in root (allow src/**/*.js if needed), .DS_Store

    8. Create drizzle.config.ts:
       - schema: "./src/db/schema/*"
       - out: "./src/db/migrations"
       - dialect: "postgresql"
       - dbCredentials from env vars (DB_HOST, DB_PORT, DB_USER, DB_PASSWORD, DB_NAME)

    9. Create src/config/env.ts:
       - Import dotenv, call config() if NODE_ENV !== 'production'
       - Validate required env vars: DB_HOST, DB_PORT, DB_NAME, DB_USER, DB_PASSWORD, SESSION_SECRET
       - Throw descriptive error listing ALL missing vars (not just first)
       - Export typed config object with database, session, redis, and nodeEnv sections
  </action>

    - `docker compose ps` shows both postgres and redis running
    - `npm install` completes without error
    - `npx tsx src/config/env.ts` runs without throwing (env validation passes)

<task type="auto">
  <name>Task 2: Create database schema, Express app skeleton, and verify startup</name>
  <files>
    src/config/database.ts
    src/db/schema/users.ts
    src/db/schema/index.ts
    src/middleware/errorHandler.ts
    src/utils/asyncHandler.ts
    src/app.ts

  <action>
    1. Create src/config/database.ts:
       - Import drizzle from 'drizzle-orm/node-postgres', pg Pool
       - Import config from env.ts and schema from db/schema/index.ts
       - Create Pool with config.database settings, max: 20, idleTimeoutMillis: 30000, connectionTimeoutMillis: 2000
       - Export `db` = drizzle(pool, { schema })
       - Export `pool` for graceful shutdown
       - Add SIGTERM handler to close pool

    2. Create src/db/schema/users.ts:
       - Use Drizzle pgTable to define 'users' table:
         - id: serial('id').primaryKey()
         - email: varchar('email', { length: 255 }).notNull().unique()
         - passwordHash: varchar('password_hash', { length: 255 }).notNull()
         - displayName: varchar('display_name', { length: 100 }).notNull()
         - role: varchar('role', { length: 20 }).notNull().default('organizer')
         - version: integer('version').notNull().default(0) -- optimistic locking
         - createdAt: timestamp('created_at', { withTimezone: true }).defaultNow().notNull()
         - updatedAt: timestamp('updated_at', { withTimezone: true }).defaultNow().notNull()
       - CRITICAL: All timestamp columns MUST use { withTimezone: true } (timestamptz)
       - CRITICAL: version field MUST exist for optimistic locking pattern
       - Export the table definition and inferred types (InferSelectModel, InferInsertModel)

    3. Create src/db/schema/index.ts:
       - Re-export everything from users.ts
       - This file will grow as new schemas are added in later phases

    4. Create src/middleware/errorHandler.ts:
       - Export errorHandler function (err, req, res, next signature)
       - In development: return status code, error message, and stack trace
       - In production: return status code and generic message for 500s, actual message for 4xx
       - Log all errors to console.error with timestamp

    5. Create src/utils/asyncHandler.ts:
       - Export asyncHandler wrapper that catches promise rejections and passes to next()
       - This replaces express-async-errors (which has ESM compatibility issues)
       - Usage: router.get('/path', asyncHandler(async (req, res) => { ... }))

    6. Create src/app.ts:
       - Import express, helmet, errorHandler
       - Create Express app
       - Apply helmet() middleware
       - Apply express.json() and express.urlencoded({ extended: true })
       - Add a health check route: GET /health -> { status: 'ok', timestamp: new Date().toISOString() }
       - Apply errorHandler as LAST middleware
       - Export app

    7. Create src/server.ts:
       - Import config from env.ts (triggers validation)
       - Import app from app.ts
       - Listen on config port (default 3000)
       - Log startup message with port number

    8. Push database schema:
       - Run `npx drizzle-kit push` to create users table in PostgreSQL
       - Verify table exists with version and timestamptz columns
  </action>
  <verify>
    - `npm run dev` starts the server on port 3000
    - `curl http://localhost:3000/health` returns { status: 'ok', timestamp: '...' }
    - `npx drizzle-kit push` creates the users table
    - Connect to PostgreSQL and verify: `docker compose exec postgres psql -U postgres -d astoria_runners -c "\d users"` shows version integer column and timestamptz columns
  </verify>
  <done>
    Express app starts and responds to /health. Users table exists in PostgreSQL with id, email, password_hash, display_name, role, version (integer, default 0), created_at (timestamptz), and updated_at (timestamptz). Drizzle ORM connects to database via connection pool.
  </done>
