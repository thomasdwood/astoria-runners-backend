  <action>
Create `src/routes/events.ts` following the exact pattern from src/routes/routes.ts:

- Apply `router.use(requireAuth)` at top (all event routes require auth)
- Import asyncHandler, requireAuth, validateBody, validateQuery
- Import event validation schemas from src/validation/events.ts
- Import eventService

Endpoints:
1. `POST /` -- Create event
   - validateBody(createEventSchema)
   - Call eventService.createEvent(req.body)
   - If result has error 'route_not_found', return 422 with `{ error: 'Route not found' }`
   - Return 201 with `{ event }`

2. `GET /` -- List events with optional filters
   - validateQuery(listEventsQuerySchema)
   - Extract category, start, end from req.query
   - Convert start/end strings to Date objects if present
   - Call eventService.listEvents({ category, start, end })
   - Return 200 with `{ events }`

3. `GET /:id` -- Get single event
   - Parse id from req.params (handle Array.isArray case, same as routes.ts)
   - Call eventService.getEventById(id)
   - Return 404 if null, 200 with `{ event }` otherwise

4. `PUT /:id` -- Update event with optimistic locking
   - validateBody(updateEventSchema)
   - Parse id (same pattern)
   - Check at least one data field beyond version
   - Call eventService.updateEvent(id, req.body)
   - Handle not_found (404), conflict (409), success (200 with `{ event }`)

5. `DELETE /:id` -- Delete event
   - Parse id (same pattern)
   - Call eventService.deleteEvent(id)
   - Handle not_found (404), success (204)

Export default router.

Create `src/routes/recurringTemplates.ts` following same pattern:

- Apply `router.use(requireAuth)` at top
- Import recurringService

Endpoints:
1. `POST /` -- Create recurring template
   - validateBody(createRecurringTemplateSchema)
   - Call recurringService.createRecurringTemplate(req.body)
   - If route_not_found error, return 422
   - Return 201 with `{ template }`

2. `GET /` -- List active recurring templates
   - Optional category query param filter
   - Call recurringService.listRecurringTemplates(filters)
   - Return 200 with `{ templates }`

3. `GET /:id` -- Get single template
   - Same id parsing pattern
   - Call recurringService.getRecurringTemplateById(id)
   - Return 404 or 200 with `{ template }`

4. `GET /:id/instances` -- Get generated instances for a template in date range
   - Parse id
   - Accept start and end query params (required ISO date strings)
   - Call recurringService.getInstancesInRange(id, new Date(start), new Date(end))
   - Return 200 with `{ instances }`

5. `PUT /:id` -- Update recurring template
   - validateBody(updateRecurringTemplateSchema)
   - Same pattern as event update (optimistic locking, 404/409/200)

6. `DELETE /:id` -- Delete/deactivate recurring template
   - Call recurringService.deleteRecurringTemplate(id)
   - Handle not_found (404), success (204)

Export default router.

Update `src/app.ts`:
- Import eventsRouter from './routes/events.js'
- Import recurringTemplatesRouter from './routes/recurringTemplates.js'
- Mount: `app.use('/api/events', eventsRouter)` -- auth handled in router
- Mount: `app.use('/api/recurring-templates', recurringTemplatesRouter)` -- auth handled in router
- Add comments consistent with existing style
