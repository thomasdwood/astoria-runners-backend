  <action>
    1. Refactor `buildRRule()` signature to accept a pattern object instead of individual params:
       ```
       interface RecurrencePattern {
         frequency: 'weekly' | 'biweekly' | 'monthly';
         dayOfWeek: number;       // 0=Sun through 6=Sat
         bySetPos?: number;       // 1-4 or -1 (last), only for monthly
         startTime: string;       // HH:MM
         endDate?: Date;          // optional until date
       }
       ```
       - Keep `computeDtstart()` and `dayOfWeekToRRuleDay()` helpers unchanged
       - New buildRRule logic:
         - weekly: `{ freq: RRule.WEEKLY, byweekday: [day], dtstart }`
         - biweekly: `{ freq: RRule.WEEKLY, interval: 2, byweekday: [day], dtstart }`
         - monthly: `{ freq: RRule.MONTHLY, byweekday: [day], bysetpos: [pattern.bySetPos!], dtstart }`
       - If `endDate` provided, add `until: endDate` to options
       - NEVER include `count` in new RRULEs — use `.between()` for generation
       - Return `new RRule(options).toString()`

    2. Add `formatRecurrenceText(pattern: RecurrencePattern): string` helper:
       - Define DAY_NAMES and ORDINALS constants locally:
         ```
         const ORDINALS: Record<number, string> = { 1: '1st', 2: '2nd', 3: '3rd', 4: '4th', [-1]: 'last' };
         const DAY_NAMES = ['Sunday', 'Monday', 'Tuesday', 'Wednesday', 'Thursday', 'Friday', 'Saturday'];
         ```
       - weekly: `"Every ${dayName}"`
       - biweekly: `"Every other ${dayName}"`
       - monthly: `"Every ${ordinal} ${dayName} of the month"`
       - Do NOT use rrule.toText() — it outputs incorrect text for bysetpos patterns
       - Export this function (frontend will call API to get preview text, or replicate logic)

    3. Add `getRecurrencePreview(pattern: RecurrencePattern): { text: string; nextDates: string[] }`:
       - text = formatRecurrenceText(pattern)
       - nextDates = generate next 3 upcoming dates using the built RRULE and `.between(now, 90daysFromNow, true).slice(0, 3)`
       - Format dates as ISO strings
       - Export this function

    4. Update `createRecurringTemplate()`:
       - Accept new fields: frequency, interval, bySetPos, endDate, startLocation
       - Build RRULE using new pattern-based buildRRule: `buildRRule({ frequency: data.frequency || 'weekly', dayOfWeek: data.dayOfWeek, bySetPos: data.bySetPos, startTime: data.startTime, endDate: data.endDate ? new Date(data.endDate) : undefined })`
       - Store frequency, interval, bySetPos, endDate, startLocation in the DB row
       - Remove hardcoded `count || 12` fallback

    5. Update `updateRecurringTemplate()`:
       - When rebuilding RRULE (if dayOfWeek, startTime, frequency, interval, bySetPos, or endDate changed): use new buildRRule with pattern object
       - Include new fields in the update operation
       - Fetch current template to fill missing values when partially updating

    6. Update `getAllInstancesInRange()`:
       - Add `excludedDates` filtering: for each template, parse `template.excludedDates` (JSON string array or null), and exclude occurrences that match any excluded date. Compare by date string (YYYY-MM-DD) to avoid timezone issues.
       - The existing `.between(rangeStart, rangeEnd, true)` already handles the date range — just add the exclusion filter after.
       - Default lookahead remains parameterized (callers pass the range)
