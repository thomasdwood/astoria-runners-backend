## Decisions Made
- Categories as FK table (not pgEnum) allows organizers to add/edit/remove categories at runtime without DB migrations
- Settings key-value store chosen over typed columns for flexibility (keys discoverable at runtime)
- RRULE strings generated without COUNT going forward; existing COUNT parameters stripped in migration step
- Category string filters in `routeService`, `eventService`, `recurringService` stubbed with TODO comments — full service layer migration to categoryId filters deferred to 03.1-02
- `endLocation` made nullable on routes (start == end location is valid when startLocation is set)
