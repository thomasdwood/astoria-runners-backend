  <name>Task 1: Create seed script and run integration tests</name>
  <files>
    src/db/seed.ts

  <action>
    1. Create src/db/seed.ts:
       - Import { db } from config/database.ts
       - Import { users } from db/schema
       - Import { hashPassword } from services/authService.ts
       - Import { pool } from config/database.ts (for graceful shutdown)

       - Define seed data:
         - Demo organizer: email "admin@astoriarunners.com", password "organizer123", displayName "Demo Organizer", role "organizer"
         - Second organizer: email "sarah@astoriarunners.com", password "organizer123", displayName "Sarah", role "organizer"

       - Seed logic:
         a. Hash passwords using hashPassword()
         b. Use upsert pattern: INSERT ... ON CONFLICT (email) DO UPDATE to make script idempotent
         c. Log each created/updated user (email + displayName, NOT password)
         d. Log "Seed complete" with count

       - Run as standalone script:
         ```
         async function main() {
           try {
             await seed();
             console.log('Seeding complete');
           } catch (error) {
             console.error('Seeding failed:', error);
             process.exit(1);
           } finally {
             await pool.end();
           }
         }
         main();
         ```

    2. Run the seed script: `npm run db:seed`

    3. Verify seed data in database:
       - `docker compose exec postgres psql -U postgres -d astoria_runners -c "SELECT id, email, display_name, role, version, created_at FROM users;"`
       - Confirm 2 users exist with role 'organizer', version 0, and timestamptz created_at

    4. Run integration verification via curl commands:
       - Start server: `npm run dev` (in background)

       a. Test login with correct credentials:
          `curl -v -X POST http://localhost:3000/auth/login -H 'Content-Type: application/json' -d '{"email":"admin@astoriarunners.com","password":"organizer123"}' -c cookies.txt`
          Expected: 200, Set-Cookie header with 'sid', response body { success: true, user: {...} }

       b. Test session persistence (GET /auth/me with session cookie):
          `curl -v http://localhost:3000/auth/me -b cookies.txt`
          Expected: 200, returns user info from session

       c. Test protected route without auth:
          `curl -v http://localhost:3000/auth/me`
          Expected: 401, { error: 'Authentication required' }

       d. Test public calendar without auth:
          `curl -v http://localhost:3000/calendar`
          Expected: 200, returns calendar placeholder

       e. Test login with wrong password:
          `curl -v -X POST http://localhost:3000/auth/login -H 'Content-Type: application/json' -d '{"email":"admin@astoriarunners.com","password":"wrongpassword"}'`
          Expected: 401, { error: 'Invalid credentials' }

       f. Test login with nonexistent email:
          `curl -v -X POST http://localhost:3000/auth/login -H 'Content-Type: application/json' -d '{"email":"nobody@test.com","password":"whatever"}'`
          Expected: 401, { error: 'Invalid credentials' } (same error -- no email enumeration)

       g. Test logout:
          `curl -v -X POST http://localhost:3000/auth/logout -b cookies.txt -c cookies.txt`
          Expected: 200, { success: true }, Set-Cookie clears 'sid'

       h. Test session destroyed after logout:
          `curl -v http://localhost:3000/auth/me -b cookies.txt`
          Expected: 401 (session destroyed)

       - Clean up cookies.txt after tests
  </action>
