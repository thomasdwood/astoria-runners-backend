**2. Create `src/db/schema/routes.ts`:**

Define pgEnum and routes table following the users.ts pattern:

```typescript
import { pgTable, pgEnum, serial, varchar, numeric, integer, timestamp } from 'drizzle-orm/pg-core';
import { InferSelectModel, InferInsertModel } from 'drizzle-orm';

export const routeCategory = pgEnum('route_category', [
  'Brewery Run',
  'Coffee Run',
  'Brunch Run',
  'Weekend',
]);

export const routes = pgTable('routes', {
  id: serial('id').primaryKey(),
  name: varchar('name', { length: 100 }).notNull(),
  distance: numeric('distance', { precision: 5, scale: 2 }).notNull(),
  category: routeCategory('category').notNull(),
  endLocation: varchar('end_location', { length: 200 }).notNull(),
  version: integer('version').notNull().default(0),
  createdAt: timestamp('created_at', { withTimezone: true }).defaultNow().notNull(),
  updatedAt: timestamp('updated_at', { withTimezone: true }).defaultNow().notNull(),
});

export type Route = InferSelectModel<typeof routes>;
export type NewRoute = InferInsertModel<typeof routes>;
```

Note: `numeric` stores distance as decimal string in Drizzle. The service layer will parse to number for API responses.

Note: version defaults to 0 matching the users table pattern from Phase 1.

**4. Create `src/validation/routes.ts`:**

```typescript
import { z } from 'zod';

const ROUTE_CATEGORIES = ['Brewery Run', 'Coffee Run', 'Brunch Run', 'Weekend'] as const;

export const createRouteSchema = z.object({
  name: z.string().min(1, 'Name is required').max(100, 'Name must be 100 characters or less'),
  distance: z.number().positive('Distance must be positive'),
  category: z.enum(ROUTE_CATEGORIES, {
    errorMap: () => ({ message: `Category must be one of: ${ROUTE_CATEGORIES.join(', ')}` }),
  }),
  endLocation: z.string().min(1, 'End location is required').max(200, 'End location must be 200 characters or less'),
});

export const updateRouteSchema = z.object({
  name: z.string().min(1, 'Name is required').max(100, 'Name must be 100 characters or less').optional(),
  distance: z.number().positive('Distance must be positive').optional(),
  category: z.enum(ROUTE_CATEGORIES, {
    errorMap: () => ({ message: `Category must be one of: ${ROUTE_CATEGORIES.join(', ')}` }),
  }).optional(),
  endLocation: z.string().min(1, 'End location is required').max(200, 'End location must be 200 characters or less').optional(),
  version: z.number().int().nonnegative('Version must be a non-negative integer'),
});
```

The `version` field is REQUIRED in updateRouteSchema (not optional) -- optimistic locking requires it.

The updateRouteSchema requires at least the version field. The route handler should additionally verify at least one data field is present before calling the service.

Also export the category array for reuse:


Also add a query params schema for list filtering:

Export inferred types:

**Create `src/services/routeService.ts`:**

Follow the authService.ts pattern (exported functions, db import from config/database.js):

```typescript
import { eq, and, sql } from 'drizzle-orm';
import { db } from '../config/database.js';
import { routes } from '../db/schema/routes.js';
import type { CreateRouteInput, UpdateRouteInput } from '../validation/routes.js';
```

Implement these functions:

**`createRoute(data: CreateRouteInput)`**
- Insert into routes table using `db.insert(routes).values({...}).returning()`
- Map `distance` to string for numeric column: `distance: String(data.distance)`
- Return the created route (first element of returning array)

**`getRouteById(id: number)`**
- Select from routes where id matches
- Return first result or null (same pattern as findUserByEmail)

**`listRoutes(filters?: { category?: string })`**
- Select all from routes
- If `filters?.category` is provided, add WHERE clause: `eq(routes.category, filters.category)`
- Order by `routes.name` ascending (alphabetical listing)
- Return array of routes

**`updateRoute(id: number, data: UpdateRouteInput)`**
- Extract `version` from data, spread remaining fields into update set
- Only include fields that are present (not undefined) in the SET clause
- Build update object dynamically: `const updateFields: Record<string, unknown> = {}`
- For each optional field (name, distance, category, endLocation), if present, add to updateFields
- Always set `updatedAt: sql\`NOW()\``
- Always set `version: sql\`${routes.version} + 1\``
- Execute: `db.update(routes).set(updateFields).where(and(eq(routes.id, id), eq(routes.version, version))).returning()`
- If returning array is empty, need to determine WHY:
  - Query `db.select().from(routes).where(eq(routes.id, id))` to check if route exists
  - If not found: return `{ error: 'not_found' as const }`
  - If found but version mismatch: return `{ error: 'conflict' as const }`
- If returning array has result: return `{ route: result[0] }`
- Use discriminated union return type: `Promise<{ route: Route } | { error: 'not_found' | 'conflict' }>`

Important: For the `distance` field, convert number to string before passing to drizzle: `distance: String(data.distance)`

**`deleteRoute(id: number)`**
- Delete from routes where id matches, using `.returning()`
- If returning array is empty: return `{ error: 'not_found' as const }`
- If successful: return `{ success: true as const }`
- Return type: `Promise<{ success: true } | { error: 'not_found' | 'conflict' }>`
- Note: In Phase 2, deletion always succeeds if route exists. Phase 3 will add foreign key constraint that causes PostgreSQL error 23503 on referenced routes. At that point, wrap in try/catch and return `{ error: 'conflict' }` for that error code.

**Response formatting helper:**

Add a `formatRoute` function that converts the drizzle row to API-friendly format:

```typescript
function formatRoute(row: Route) {
  return {
    ...row,
    distance: Number(row.distance), // numeric column returns string, convert to number for API
  };
}
```

Apply `formatRoute` to all return values from the service functions, so consumers always get `distance` as a number.
