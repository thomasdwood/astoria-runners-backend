# Milestone v1.0: MVP

**Status:** ✅ SHIPPED 2026-03-15
**Phases:** 01 → 02 → 03 → 03.1 → 04 → 05 → 06 → 07 → 08
**Total Plans:** 33

## Overview

Initial MVP of the Astoria Runners Planning Tool. Replaced a broken Google Sheets workflow with a full web application: organizer auth, route library, event scheduling (one-off + complex recurring), public calendar, Discord notifications, Meetup export, host assignment, security hardening, and 47 real Astoria Runners routes seeded.

## Milestone Summary

### Decimal Phases

- Phase 03.1: Refinements & Missing Features (INSERTED after Phase 3 — accumulated TODOs from initial build)

### Key Decisions

- Categories as FK table not pgEnum — runtime CRUD without migrations
- Virtual recurring instances not materialized — generated on-the-fly from RRULE strings
- postedToMeetup boolean → meetupUrl varchar — URL presence as posted status indicator
- sameSite: lax CSRF posture — sufficient for single-origin app, csrf-csrf deferred
- Wrote DB migrations manually — drizzle-kit ESM/CJS incompatibility in Phase 05
- Router-level auth (router.use) pattern established in Phase 02, applied consistently throughout
- asyncHandler utility (not express-async-errors) for ESM compatibility

### Issues Resolved

- Dynamic categories replacing hardcoded pgEnum (Phase 03.1)
- Complex recurring patterns (nth-weekday via bysetpos) — rrule.toText() insufficient, custom text formatter built
- {{distance}} variable blank in Meetup templates for virtual instances (Phase 07)
- Host edit UI wired — useUpdateHost hook was orphaned until Phase 07

### Issues Deferred (Tech Debt)

- EXPORT-01 backend template wiring: meetupExportService doesn't read stored meetup_description_template for DB events (client-side path works correctly for virtual instances)
- No test framework installed (Vitest/Jest) — zero automated tests
- Phase 01 runtime Docker verification deferred (12 curl tests never run; implicit validation via 8 subsequent phases)
- requireGuest middleware exported but never applied to /login route (minor UX, no security impact)

### Technical Debt Incurred

- DB schema applied manually for Phase 05 (not via drizzle-kit push) — migration history may diverge
- No Nyquist-compliant phases (test runner prerequisite missing)
