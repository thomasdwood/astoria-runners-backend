### Integration Gap Detail

**EXPORT-01 backend template wiring gap**

- **Affected file:** `src/services/meetupExportService.ts`
- **Problem:** `generateMeetupDescription()` generates descriptions from hardcoded `generatePlainDescription()`/`generateHtmlDescription()` functions. It never calls `getSetting('meetup_description_template')`. No `getSetting` import exists in this file.
- **Trigger:** Organizer configures custom template in Settings page → opens Meetup popover on a one-off DB event → clicks Plain or HTML format → backend endpoint `GET /api/events/:id/meetup-description` returns hardcoded format, not the custom template
- **Working path:** Virtual recurring instances use client-side `generateClientSideDescription()` → reads `meetup_description_template` via `useSettings()` → applies custom template with all variables including `{{distance}}` (Phase 07 fix confirmed)
- **Workaround:** Materialize the instance first (click Save URL → creates DB record), but then popover switches to the DB event path which also uses hardcoded format
- **Severity:** Tech debt — the feature works (descriptions can be generated and copied), but custom templates are silently ignored for DB events
- **Affected requirement:** EXPORT-01 (partially satisfied)

**`requireGuest` unused**

- **File:** `src/middleware/auth.ts` (exported), never imported in any route file
- **Impact:** Authenticated users can visit `/login` (minor UX — after submitting they redirect to `/admin`). Not a security issue.
- **Severity:** Low — minor UX polish item
